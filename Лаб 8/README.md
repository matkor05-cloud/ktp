# Лабораторная работа №8: Аннотации, Reflection API и Многопоточность

## Описание проекта
Данный проект представляет собой многопоточную систему обработки данных, построенную на основе пользовательских аннотаций и механизма рефлексии. В качестве примера реализован **конвертер валютных транзакций**, который фильтрует некорректные данные и пересчитывает суммы в другую валюту в параллельном режиме.

Основная идея проекта — создание «умного» менеджера данных, который автоматически находит методы обработки, помеченные специальной «меткой», и запускает их асинхронно.

## Используемые технологии
*   **Java 17+**
*   **Reflection API** — для динамического поиска и вызова методов во время выполнения.
*   **Custom Annotations** — для маркировки методов обработки.
*   **Stream API** — для эффективной фильтрации и трансформации коллекций.
*   **Java Concurrency (`CompletableFuture`, `ExecutorService`)** — для реализации многопоточности.

## Структура проекта
*   `DataProcessor.java` — Пользовательская аннотация `@DataProcessor` с временем жизни `RUNTIME`.
*   `CurrencyHandlers.java` — Класс, содержащий логику обработки (фильтрация отрицательных сумм и конвертация в рубли).
*   `DataManager.java` — Центральный процессор. Использует рефлексию для поиска аннотированных методов и пул потоков для их выполнения.
*   `Main.java` — Точка входа в программу: регистрация обработчиков и запуск конвейера.

## Как это работает
1.  **Маркировка:** Мы помечаем методы обработки аннотацией `@DataProcessor`.
2.  **Регистрация:** Объекты с логикой регистрируются в `DataManager`.
3.  **Инспекция (Reflection):** `DataManager` сканирует методы зарегистрированных объектов. Если метод помечен аннотацией, он берется в работу.
4.  **Параллелизм:** Каждая задача запускается в отдельном потоке через `CompletableFuture.supplyAsync`.
5.  **Стримы:** Внутри методов данные обрабатываются через `stream()`, что гарантирует чистоту и краткость кода.

## Пример работы программы
Программа принимает на вход список транзакций в долларах: `[10.0, 50.0, -5.0, 100.0]`.
После обработки:
1.  Метод фильтрации удаляет `-5.0`.
2.  Метод конвертации умножает оставшиеся суммы на курс (90.0).
3.  В итоге выводится список в рублях и общая сумма всех операций.
